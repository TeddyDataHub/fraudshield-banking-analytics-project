This folder contains the cleaned and analysis-ready fradushield-banking dataset.
The data has been processed from the raw version using Excel to improve quality and consistency for further analysis in MySQL, and Power BI.

Cleaning steps may include:

handling missing values
removing duplicate records
standardizing column names and formats
correcting data inconsistencies
validating transaction fields
preparing fields for pivot tables and dashboard reporting

🎯 Purpose

The cleaned dataset is used to:

support accurate fraud analysis
identify suspicious transaction patterns
enable SQL querying
create pivot tables and dashboards
prepare data for Power BI visualization
